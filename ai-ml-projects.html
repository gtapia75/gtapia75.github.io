Video Retrieval-Augmented Generation (Video RAG) with AWS & OpenAI

Strategic Purpose: Explored how multimodal AI and retrieval systems can help organizations search, understand, and reason over video assets without relying entirely on manual tagging or metadata.

System Design:

  • Processed video files from Amazon S3 and extracted representative timestamped frames using OpenCV.
  • Generated multimodal embeddings with Amazon Bedrock Titan to map visual content and natural language queries into a shared semantic space.
  • Stored embeddings in ChromaDB to support vector similarity search and timestamp-based retrieval.
  • Integrated optional OpenAI vision capabilities to explain retrieved frames and provide human-readable context.

Enterprise Relevance:

  • Demonstrates how AI can improve discovery across large media libraries, content archives, production assets, and operational video workflows.
  • Supports use cases such as scene search, compliance review, content classification, media operations, and video intelligence.
  • Reduces dependency on manual tagging by enabling natural language search across visual assets.

What This Proves:

  • Multimodal retrieval can make unstructured video content searchable and operationally useful.
  • Frame-level indexing creates a practical bridge between video assets and AI-assisted discovery systems.
  • The architecture can be extended toward long-form video understanding, audio analysis, metadata enrichment, and enterprise media intelligence.

URL Similarity Search System using ChromaDB

Overview: This project implements a smart URL matching system that helps users find the correct webpage even when they encounter typos or slightly incorrect URLs. By converting URLs into numerical vectors and using ChromaDB for similarity search, the system identifies and suggests the most similar valid URLs from a website's sitemap.

Features:

  • Processes XML sitemaps to extract URLs
  • Converts URLs into vector representations
  • Stores these vectors in ChromaDB for efficient similarity search
  • Finds similar URLs when given a potentially incorrect URL
  • Redirects users to the closest matching content

Challenges & Complexity:

  • Ensuring accuracy when processing large amounts of URL data
  • Developing vectorization techniques for URL comparison
  • Handling multiple URL structures, formats, and patterns

Impact & Achievements:

  • Improved user experience by guiding users to the correct content instead of showing generic 404 errors
  • Reduced user frustration and maintained engagement on the site
  • Enabled support for content migration, legacy URL structures, and preserved SEO value
