Video Retrieval-Augmented Generation (Video RAG) with AWS & OpenAI

Overview: Designed and implemented a multimodal Video RAG proof-of-concept that enables semantic search over video content using natural language queries. The system processes short videos stored in Amazon S3 by extracting representative frames, generating multimodal embeddings with Amazon Bedrock Titan models, and storing them in a vector database for efficient retrieval. The solution optionally integrates OpenAI vision models to generate human-readable explanations of retrieved video segments.

Features:

  • Processes video files from Amazon S3 and extracts timestamped frames using OpenCV.
  • Generates multimodal embeddings (text / image shared space) with Amazon Bedrock Titan.
  • Stores frame embeddings in ChromaDB to support similarity search at scale.
  • Enables text-based queries such as "find scenes with a beach" or "show moments with cars".
  • Returns relevant video frames along with their corresponding timestamps.
  • Optional OpenAI vision integration to summarize or explain retrieved frames.

Challenges & Complexity:

  • Balancing semantic coverage and cost through configurable frame sampling strategies.
  • Designing stable frame identifiers to avoid redundant embeddings and control API usage.
  • Handling corrupted frames or partial failures without interrupting ingestion.
  • Structuring the pipeline to be notebook-friendly while remaining portable to production architectures.

Impact & Achievements:

  • Demonstrated effective semantic search across video content without manual tagging.
  • Enabled precise localization of relevant moments within videos using timestamps.
  • Established a reusable multimodal pipeline that can be extended to audio analysis, scene detection, and long-form video understanding.
  • Serves as a foundation for scalable video intelligence applications and advanced RAG systems.

Multimodal Image Retrieval (RAG) with AWS & OpenAI

Overview: Built a state-of-the-art multimodal Retrieval-Augmented Generation (RAG) system that enables semantic image search from a large collection of photos stored in Amazon S3. The system uses Amazon Bedrock Titan multimodal embeddings to map images and text into a shared vector space, stores them in ChromaDB, and returns the most relevant images for a user's natural language query. Optionally, the system generates descriptive summaries of results using OpenAI Vision models.

Features:

  • Downloads and normalizes images (JPEG/PNG) from an S3 bucket.
  • Generates robust multimodal embeddings (shared space for text & images) using Amazon Bedrock Titan models.
  • Stores vectors in a persistent vector database (ChromaDB) for scalable semantic search.
  • Supports natural language queries such as "Find images with a blue car" or "Show cars on the beach".
  • Optional integration with OpenAI Vision models to produce human-readable summaries referencing labeled image results.

Challenges & Complexity:

  • Handling a heterogeneous image dataset and preventing embedding model failures via normalization.
  • Designing efficient batching and caching strategies to control API usage costs.
  • Building a retrieval interface that gracefully handles ambiguous or partial queries.

Impact & Achievements:

  • Demonstrated seamless semantic image search across thousands of images using simple natural language.
  • Reduced false matches and improved relevance through multimodal embeddings.
  • Enabled interpretable outputs by combining retrieval with generative vision summaries.
  • Provides a foundation for extending the same pipeline to video RAG and multimodal search apps.

URL Similarity Search System using ChromaDB

Overview: This project implements a smart URL matching system that helps users find the correct webpage even when they encounter typos or slightly incorrect URLs. By converting URLs into numerical vectors and using ChromaDB for similarity search, the system identifies and suggests the most similar valid URLs from a website's sitemap.

Features:

  • Processes XML sitemaps to extract URLs
  • Converts URLs into vector representations
  • Stores these vectors in ChromaDB for efficient similarity search
  • Finds similar URLs when given a potentially incorrect URL
  • Redirects users to the closest matching content

Challenges & Complexity:

  • Ensuring accuracy when processing large amounts of URL data
  • Developing vectorization techniques for URL comparison
  • Handling multiple URL structures, formats, and patterns

Impact & Achievements:

  • Improved user experience by guiding users to the correct content instead of showing generic 404 errors
  • Reduced user frustration and maintained engagement on the site
  • Enabled support for content migration, legacy URL structures, and preserved SEO value
